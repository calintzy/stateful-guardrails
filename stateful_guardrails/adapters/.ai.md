# adapters — AI 가이드

> 이 디렉토리를 수정하기 전에 반드시 이 파일을 읽는다.

## 이 디렉토리의 역할

외부 경계 레이어. LLM provider(OpenAI/Anthropic/Ollama), 메모리 스토어, 영속 상태 저장소(SQLite/JSON), 감사 로그 싱크를 담당한다. core의 Protocol을 구현하고, pipeline이 이를 주입받아 사용한다.

## 허용되는 것

- `stateful_guardrails.core.*` import
- `httpx`, `sqlite3`, `json`, `pathlib` 등 외부 I/O 라이브러리
- 파이썬 표준 라이브러리

## 금지되는 것

- `stateful_guardrails.interfaces.*` import 금지
- `stateful_guardrails.pipeline.*` import 금지 (순환 의존 방지)
- 비즈니스 로직(판정·에스컬레이션 결정) 작성 금지 — pipeline/core의 책임

## 파일 구조

```
adapters/
  __init__.py     ← 레이어 설명
  llm.py          ← OllamaAdapter, OpenAIAdapter, AnthropicAdapter + get_adapter()
  embeddings.py   ← OllamaEmbedder (bge-m3 임베딩 어댑터)
  state_store.py  ← JSONStateStore 영속화 어댑터 (JSON 파일) (Phase 2~)
  [Phase 4 미구현] audit.py ← 감사 로그 싱크 어댑터
```

## 주의사항

- `get_adapter(provider)` 팩토리를 통해 인스턴스를 제공한다. interfaces/pipeline은 구체 클래스를 직접 import하지 않는 것을 권장한다.
- Ollama 기본 모델은 `qwen2.5:14b`. bge-m3는 임베딩 전용이므로 `complete()` 호출에 사용하지 않는다.
- Phase 0에서는 OllamaAdapter만 완전 구현. OpenAI/Anthropic는 `NotImplementedError` 스텁.
