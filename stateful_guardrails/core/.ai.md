# core — AI 가이드

> 이 디렉토리를 수정하기 전에 반드시 이 파일을 읽는다.

## 이 디렉토리의 역할

순수 도메인 레이어. Policy 인터페이스, 위험 판정 로직, 세션 상태 모델, 에스컬레이션 정책, 데이터셋 스키마를 담는다. 외부 I/O·프레임워크·다른 레이어에 의존하지 않으므로 테스트가 가장 쉽다.

## 허용되는 것

- 파이썬 표준 라이브러리만 import
- `typing`, `dataclasses`, `enum`, `abc`, `pathlib`, `datetime` 등 stdlib
- `__future__` import

## 금지되는 것

- `stateful_guardrails.adapters.*` import 절대 금지
- `stateful_guardrails.pipeline.*` import 절대 금지
- `stateful_guardrails.interfaces.*` import 절대 금지
- `langgraph`, `langchain`, `httpx`, `fastapi` 등 외부 라이브러리 직접 import 금지
- 파일 I/O, 네트워크 호출, DB 접근 금지 (어댑터를 통해서만)

## 파일 구조

```
core/
  __init__.py    ← 레이어 설명 + 공개 인터페이스 re-export
  llm.py         ← LLMAdapter Protocol (추상 인터페이스)
  policy.py      ← Policy Protocol, PolicySignal, RiskCategory, SessionState (Phase 1~)
  state.py       ← SessionState 도메인 모델 (Phase 2~)
  vectormath.py  ← 코사인 유사도·노름 등 순수 벡터 연산
```

## 코드 패턴

```python
# 올바른 예시 — Protocol로 인터페이스 정의, 구현은 adapters에
from typing import Protocol, runtime_checkable

@runtime_checkable
class LLMAdapter(Protocol):
    def complete(self, prompt: str) -> str: ...
```

```python
# 잘못된 예시 — core에서 직접 구현체 import
from stateful_guardrails.adapters.llm import OllamaAdapter  # 금지!
import httpx  # 금지!
```

## 주의사항

- ISC-0.2 자동 검증(`pytest tests/test_layering.py`)이 이 레이어의 import를 정적으로 감시한다.
- Protocol의 `id`, `category` 등 필드는 재번호·재명명 금지 (리포트 키로 쓰임).
- 감쇠 공식 `S_t = clip(λ·S_{t-1} + signal_t, 0, S_max)` 는 core에서 순수 함수로 구현한다 (I/O 없음).
