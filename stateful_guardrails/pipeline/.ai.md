# pipeline — AI 가이드

> 이 디렉토리를 수정하기 전에 반드시 이 파일을 읽는다.

## 이 디렉토리의 역할

판정 파이프라인 오케스트레이션 레이어. 메시지 인입 → 정책 실행 → 에스컬레이션 결정 → 감사 로그 흐름을 조율한다. LangGraph 노드로 끼는 어댑터도 여기에 위치한다.

## 허용되는 것

- `stateful_guardrails.core.*` import
- `stateful_guardrails.adapters.*` import (외부 I/O는 어댑터를 통해서만)
- 파이썬 표준 라이브러리

## 금지되는 것

- `stateful_guardrails.interfaces.*` import 금지 (의존 방향 위반 — interfaces가 pipeline을 호출하지, 반대는 안 됨)
- 직접 httpx/requests 호출 금지 (adapters를 통해서만)
- 직접 DB 쿼리 금지 (adapters를 통해서만)

## 파일 구조

```
pipeline/
  __init__.py   ← 레이어 설명
  engine.py     ← 판정 엔진 (stateless / stateful 경로 분기) (Phase 1~)
  catalog.py    ← 정책 카탈로그 + 팩토리 (get_stateless_policies 등)
  policies.py   ← 구체 Policy 구현체 (keyword, semantic, pattern 등)
  scanner.py    ← stateless 스캔 + FPR 캘리브레이션 (ISC-1.4)
  detectors.py  ← target-aware/agnostic 검출기 (임베딩 기반)
  eval.py       ← 평가·메트릭 산출 (test-split 사용 금지 — ISC-2.6)
  [Phase 4 미구현] escalation.py ← 에스컬레이션 3단계 결정 로직
  [Phase 4 미구현] audit.py      ← 감사 로그 기록
```

```python
def run_pipeline(message, adapter: OllamaAdapter, policy: Policy): ...
```

```python

```

## 주의사항

- `session_state=None` 경로가 B1(stateless) baseline, 상태 참조 경로가 STATEFUL. 두 경로는 동일 코드를 통해야 공정한 비교가 가능하다.
- 에스컬레이션 결정은 결정적(deterministic)이어야 감사 로그가 재현 가능하다. LLM 자율 결정 금지.
